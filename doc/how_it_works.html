How the Code Works

There are three co-operating processes in this code.

  1. The hub process (Erlang), which communicates with the other two processes.

  2. The Java process. It gets a list of things to draw from the hub, and sends the drawing environment to the hub.

  3. The sketch process (Erlang). It sends the hub things that it wants to draw, and awaits the hub’s response. This is necessary to make sure that calls to the hub stay in order.
